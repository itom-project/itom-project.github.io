Basic Filters¶

Summary: ITO developed filter-functions for dateObjects
Type: Algorithm
License: LGPL
Platforms: Windows, Linux
Author: W. Lyda, T. Boettcher, University Stuttgart

Overview¶

This plugin provides several basic filter calculations for itom::dataObject. These are for instance:

  • merging of planes
  • swap byte order of objects
  • resample slices from dataObjects
  • mean value filter along axis

This plugin does not have any unusual dependencies.

These filters are defined in the plugin:

  1. highValueFilter
  2. replaceInfAndNaN
  3. mergeColorPlane
  4. calcHist
  5. lowValueFilter
  6. swapByteOrder
  7. lowPassFilter
  8. clipValues
  9. flatten3Dto2D
  10. calcObjSlice
  11. medianFilter
  12. calcMeanZ
  13. clipAbyB

Filters¶

Detailed overview about all defined filters:

itom.filter("highValueFilter",sourceImage, destImage, kernelx, kernely[, replaceNaN])¶

Set each pixel to the highest value within the kernel (x ,y) using the generic mcpp filter engine

Mandatory parameters:
 
  • sourceImage (dataObject) – n-dim DataObject
  • destImage (dataObject) – n-dim DataObject of type sourceImage
  • kernelx (int (int)) – Odd kernelsize in x
  • kernely (int (int)) – Odd kernelsize in y
Optional parameters:
 

replaceNaN (int (int)) – if 0 NaN values in input image will be copied to output image (default)
itom.filter("replaceInfAndNaN",srcImg, replaceImg, destImg)¶

replaces infinite and/or nan-values by values of second matrix.

Mandatory parameters:
 
  • srcImg (dataObject) – Input image
  • replaceImg (dataObject) – Image with values which will be used for replacement
  • destImg (dataObject) – Output image
Returns:

nrOfReplacements (int (int)) – number of replacments
itom.filter("mergeColorPlane",srcImg, destImg[, toogleByteOrder])¶

Merge 3 or 4 color planes to a single tRGBA32 or tInt32-plane.

If second object is tInt32 and of right size in x and y, the stack object will be convertet to tInt32. In all other cases the object will be tRGBA32

Mandatory parameters:
 
  • srcImg (dataObject) – Input image with 3 or 4 uint8 planes
  • destImg (dataObject) – Output image with uint32 planes
Optional parameters:
 

toogleByteOrder (int (int)) – Switch between RGBA = 0, BGRA = 1, ARGB = 2, ABGR = 3

  • sourceImage (dataObject) – Image of type Integer or float

itom.filter("clipValues",sourceImage, destinationImage, minValue, maxValue[, newValue, insideFlag])¶

clip values outside or inside of minValue and maxValue to newValue (default = 0)

Depending on the parameter ‘insideFlag’, this filter sets all values within (1) or outside (0) of the range (minValue, maxValue) to the value given by ‘newValue’. In both cases the range boundaries are not clipped and replaced. If clipping is executed outside of range, NaN and Inf values are replaced as well (floating point data objects only). This filter supports only real value data types.

Mandatory parameters:
 
  • sourceImage (dataObject) – input image [real typed data object]
  • destinationImage (dataObject) – destination image (inplace possible)
  • minValue (float (double)) – lowest value in range
  • maxValue (float (double)) – highest value in range
Optional parameters:
 
  • newValue (float (double)) – value set to clipped values (default: 0.0)
  • insideFlag (int (int)) – 0: clip values outside of given range (default), 1: clip inside

Mandatory parameters:

  • dstSlice (dataObject) – Slice with output data
  • x0 (float (double)) – x0-coordinate for slice
  • y0 (float (double)) – y0-coordinate for slice
  • x1 (float (double)) – x1-coordinate for slice
  • y1 (float (double)) – y1-coordinate for slice
Optional parameters:
 

mode (interpolation) – 0: Bresenham or Nearest, 1: weighted

itom.filter("clipAbyB",sourceImage, comparisonImage, destinationImage, minValue, maxValue[, newValue, insideFlag])¶

clip values of image A to newValue (default = 0) outside or inside of minValue and maxValue in image B

Depending on the parameter ‘insideFlag’, this filter sets all values in image A depending on image B within (1) or outside (0) of the range (minValue, maxValue) to the value given by ‘newValue’. In both cases the range boundaries are not clipped and replaced. If clipping is executed outside of range, NaN and Inf values are replaced as well (floating point data objects only). This filter supports only real value data types.

Mandatory parameters:
 
  • sourceImage (dataObject) – input image [real typed data object]
  • comparisonImage (dataObject) – input image [real typed data object] for comparision
  • destinationImage (dataObject) – destination image (inplace possible)
  • minValue (float (double)) – lowest value in range
  • maxValue (float (double)) – highest value in range
Optional parameters:
 
  • newValue (float (double)) – value set to clipped values (default: 0.0)
  • insideFlag (int (int)) – 0: clip values outside of given range (default), 1: clip inside
