DataObject Arithmetic¶

Summary: Operations and arithmetic calculations of dataObject.
Type: Algorithm
License: LGPL
Platforms: Windows, Linux
Author: W. Lyda, M. Gronle, ITO, University Stuttgart

Overview¶

This plugin provides several arithmetic calculations for dataObject. These are for instance: - min- or maximum value - centroid along dimensions or inplane

This plugin does not have any unusual dependencies.

These filters are defined in the plugin:

  1. centroid1D
  2. meanValue
  3. deviationValue
  4. maxValue
  5. minValue
  6. getPercentageThreshold
  7. centroidXY
  8. areEqual
  9. minMaxValue

Filters¶

Detailed overview about all defined filters:

itom.filter("centroid1D",sourceStack, cogPlane, intPlane[, lowTreshold, dynamicTreshold, columnWise])¶

This filter calculates the center of gravity along y or x direction.

The filter do not work with RGBA32, Complex64 and Complex128, but with all other data-types

Mandatory parameters:
 
  • sourceStack (dataObject) – source image data object for operation
  • cogPlane (dataObject) – destination image data object for operation, will contain evaluated COG
  • intPlane (dataObject) – destination image data object for operation, will contain maximal Intensity
Optional parameters:
 
  • lowTreshold (float (double)) – Ingnore values lower than lowTreshold
  • dynamicTreshold (float (double)) – Dynamic treshold, e.g. half maximum
  • columnWise (int (int)) – Evaluate along Columns
itom.filter("meanValue",sourceImage[, ignoreInf])¶

This filter calculated the mean value within the dataObject.

The return value containing the mean value of the dataObject.

The filter do not work with RGBA32, Complex64 and Complex128, but with all other data-types

Mandatory parameters:
 sourceImage (dataObject) – source image data object for operation
Optional parameters:
 ignoreInf (int (int)) – source image data object for operation
Returns:result (float (double)) – result of calculation. This param can be int or double
itom.filter("deviationValue",sourceImage[, flag, ignoreInf])¶

The filter returns the arithmetic mean and the standard deviation of the given dataObject within its ROI. The optinal flag to toggles if (flag==0) the deviation is calculated by 1/(n-1)*sqrt(sum(x-xm)^2) or (flag ==1) by 1/(n)*sqrt(sum(x-xm)^2)

The filter do not work with RGBA32, Complex64 and Complex128, but with all other data-types

Mandatory parameters:
 

sourceImage (dataObject) – source image data object for operation
Optional parameters:
 
  • flag (int (int)) – Toggles the calculation mode of standard deviation over N or N-1 elements
  • ignoreInf (int (int)) – Ignore invalid-Values for floating point
Returns:
  • mean (float (double)) – mean result
  • dev (float (double)) – deviation result
itom.filter("maxValue",sourceImage[, ignoreInf])¶

This filter calculated the maximal value and its first location within the dataObject.

The result value will be Integer vor all integer types or Double for all floating point types

The filter do not work with RGBA32, Complex64 and Complex128, but with all other data-types

Mandatory parameters:
 

sourceImage (dataObject) – source image data object for operation
Optional parameters:
 

ignoreInf (int (int)) – Ignore invalid-Values for floating point
Returns:
  • result (float (double)) – result of calculation. This param can be int or double
  • plane (int (int)) – Index of the plane, which contains the result.
  • y (int (int)) – Pixelindex in y-direction.
  • x (int (int)) – Pixelindex in x-direction.

itom.filter("getPercentageThreshold",data, percentage)¶

analyzes all values in the given data object and returns the value, which is at a given percentage in the sorted value list.

Mandatory parameters:
 
  • data (dataObject) – valid non-complex data object
  • percentage (float (double)) – percentage value [0.0,100.0]
Returns:

threshold (float (double)) – threshold value (NaN if data object was empty or only contained invalid values)
itom.filter("centroidXY",sourceImage[, lowTreshold, highTreshold])¶

This filter calculates the center of gravity within a image plane. The return value contains the column and row position in pixel and physical coordinates.

The filter do not work with RGBA32, Complex64 and Complex128, but with all other data-types.

Mandatory parameters:
 

sourceImage (dataObject) – source image data object for operation
Optional parameters:
 
  • lowTreshold (float (double)) – Ingnore values lower than lowTreshold
  • highTreshold (float (double)) – Ingnore values above highTreshold
Returns:
  • cYI (float (double)) – y-Coordinate of COG (index)
  • cXI (float (double)) – x-Coordinate of COG (index)
  • cY (float (double)) – y-Coordinate of COG (unit)
  • cX (float (double)) – x-Coordinate of COG (unit)
itom.filter("areEqual",sourceImage1, sourceImage2)¶

Check pixel-wise wether two dataObjects are equal. The filter returns 1 if both objects are pixel-wise equal, else returns 0.

The filter do not work with RGBA32, Complex64 and Complex128, but with all other data-types

Mandatory parameters:
 
  • sourceImage1 (dataObject) –
    1. source image data object for operation
  • sourceImage2 (dataObject) –
    1. source image data object for operation
Returns:

result (int (int)) – 0 if both data objects are not equal, else 1
itom.filter("minMaxValue",sourceImage[, ignoreInf, complexHandling])¶

This filter calculated the minimal and maximal value and its first location within the dataObject.

The result value will be Integer vor all integer types or Double for all floating point types

The filter do not work with RGBA32 but with all other data-types

Mandatory parameters:
 

sourceImage (dataObject) – source image data object for operation
Optional parameters:
 
  • ignoreInf (int (int)) – Ignore invalid-Values for floating point
  • complexHandling (int (int)) – Switch complex handling, 0:abs-Value, 1:imaginary-Value, 2:real-Value, 3: argument-Value
Returns:
  • minimum (float (double)) – Minimal value, this parameter be int or double
  • planeMin (int (int)) – Index of the plane, which contains the result.
  • yMin (int (int)) – Pixelindex in y-direction.
  • xMin (int (int)) – Pixelindex in x-direction.
  • maximum (float (double)) – Maximum value, this parameter. This param can be int or double
  • planeMax (int (int)) – Index of the plane, which contains the result.
  • yMax (int (int)) – Pixelindex in y-direction.
  • xMax (int (int)) – Pixelindex in x-direction.
