PclTools¶

Summary: Filters and methods for pointClouds and polygonMeshes
Type: Algorithm
License: licensed under LGPL
Platforms: Windows, Linux
Author: M. Gronle, ITO, University Stuttgart

Overview¶

This plugin contains methods for filtering, transforming, saving and loading point clouds and polygon meshes. Most methods are wrappers for functions provided by the open source project PointCloudLibrary (pcl). The function calls are usually implemented for the cloud types supported by the itom classes itom.pointCloud and itom.polygonMesh (XYZ,XYZI,XYZRGBA,XYZNormals...).

This library uses also methods from the current pcl version 1.7.1, however also compiles with older versions. In this case some methods are not compiled.

This plugin also covers the methods for loading and saving point clouds and polygon meshes to common formats like pcd, ply, stl, obj... Once the plugin is loaded itom in general is also able to load and save such structures using the methods provided by this plugin.

These filters are defined in the plugin:

  1. pclRemoveNaN
  2. pclPolygonMeshFromIndices
  3. loadPointCloud
  4. loadPolygonMesh
  5. pclVoxelGrid
  6. pclRandomSample
  7. pclCylinderClipper3D
  8. savePointCloud
  9. pclPCA
  10. pclFitCylinder
  11. pclGetHistogram
  12. savePolygonMesh
  13. pclGetMinMax3D
  14. pclEstimateNormals
  15. pclTransformAffine
  16. pclStatisticalOutlierRemoval
  17. pclPassThrough
  18. pclGetPercentageThreshold
  19. pclMeshTriangulation

Filters¶

Detailed overview about all defined filters:

itom.filter("pclRemoveNaN",pointCloudIn, pointCloudOut)¶

removes NaN values from input point cloud (input and output can be the same).

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • pointCloudOut (pointCloud) – Output point cloud with removed NaN values
itom.filter("pclPolygonMeshFromIndices",meshIn, meshOut, indices)¶

get sub-mesh from given mesh by the indices of the polygons.

Mandatory parameters:
 
  • meshIn (polygonMesh) – Valid polygon mesh
  • meshOut (polygonMesh) – output polygon mesh
  • indices (int seq. (int*)) – vector with indices of polygons that will be copied into output mesh
itom.filter("loadPointCloud",pointCloud, filename[, type])¶

loads pointCloud from hard drive and returns it (format pcd(binary or ascii), ply(binary or ascii), vtk(ascii)

Mandatory parameters:
 
  • pointCloud (pointCloud) – loaded pointcloud
  • filename (str (char*)) – complete filename (type is read by suffix)
Optional parameters:
 

type (str (char*)) – type (‘pcd’,’ply’,’vtk’,’auto’ [default, check suffix of filename])
itom.filter("loadPolygonMesh",polygonMesh, filename[, type])¶

loads polygonMesh from hard drive and returns it (format obj[default], ply, vtk, stl)

Mandatory parameters:
 
  • polygonMesh (polygonMesh) – loaded polygon mesh
  • filename (str (char*)) – complete filename (type is read by suffix
Optional parameters:
 

type (str (char*)) – type (‘obj’,’vtk’,’stl’,’ply’, ‘auto’ [default, check suffix of filename])
itom.filter("pclVoxelGrid",pointCloudIn, pointCloudOut, leafSize[, downsampleAllData, fieldName, fieldMin, fieldMax, negative])¶

downsamples a point cloud using a voxelized gripd approach.

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • pointCloudOut (pointCloud) – Output point cloud with removed NaN values
  • leafSize (float seq. (double*)) – voxel grid leaf size [lx,ly,lz] (3 elements necessary)
Optional parameters:
 
  • downsampleAllData (int (int)) – 1: downsample all fields, 0: only xyz [default]
  • fieldName (str (char*)) – field name, whose value is filtered between ‘fieldMin’ and ‘fieldMax’ or ‘’ if no field filtering should be applied [default]
  • fieldMin (float (double)) – minimum field filtering value (default: FLT_MIN)
  • fieldMax (float (double)) – maximum field filtering value (default: FLT_MAX)
  • negative (int (int)) – 0 [default]: values inside of field range will be deleted, else 1
itom.filter("pclRandomSample",pointCloudIn, pointCloudOut, nrOfPoints)¶

randomly reduces the point cloud to a given number of points.

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • pointCloudOut (pointCloud) – Output point cloud with removed NaN values
  • nrOfPoints (int (int)) – number of randomly picked points
itom.filter("pclCylinderClipper3D",pointCloudIn, pointCloudOut, point, orientation, radius)¶

clips points from the point cloud that lie outside a given cylindrical tube.

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • pointCloudOut (pointCloud) – Output point cloud
  • point (float seq. (double*)) – point on axis of cylinder
  • orientation (float seq. (double*)) – orientation vector of cylinder
  • radius (float seq. (double*)) – array with [min,max] radius
itom.filter("savePointCloud",pointCloud, filename[, mode, type])¶

saves pointCloud to hard drive (format pcd(binary or ascii), ply(binary or ascii), vtk(ascii), xyz(ascii)

Mandatory parameters:
 
  • pointCloud (pointCloud) – point cloud to save
  • filename (str (char*)) – complete filename (type is either read by suffix of filename or by parameter ‘type’
Optional parameters:
 
  • mode (str (char*)) – mode (b=binary (default) or t=ascii, for type ‘pcd’ and ‘ply’ only)
  • type (str (char*)) – type (‘pcd’ [default],’ply’,’vtk’,’xyz’)
itom.filter("pclPCA",pointCloudIn[, eigenVectors])¶

determines PCA of point Cloud.

Mandatory parameters:
 

pointCloudIn (pointCloud) – Valid input point cloud
Optional parameters:
 

eigenVectors (dataObject) – results in 3x3 float32 data object with three eigen-vectors
Returns:
  • mean (float seq. (double*)) – mean value
  • eigenValues (float seq. (double*)) – eigen values
itom.filter("pclFitCylinder",pointCloudIn, radiusLimits[, normalDistanceWeight, maxIterations, distanceThreshold, optimizeParameters, probability])¶

fits a cylindrical model to the given input point cloud using a RANSAC based fit (must have normals defined).

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Input point cloud with normal values
  • radiusLimits (float seq. (double*)) – radius limits [min, max]
Optional parameters:
 
  • normalDistanceWeight (float (double)) – Set the relative weight (between 0 and 1) to give to the angular distance (0 to pi/2) between point normals and the plane normal [default: 0.1]
  • maxIterations (int (int)) – maximum number of RANSAC iterations [default: 10000]
  • distanceThreshold (float (double)) – distanceThreshold of pcl [default: 0.05]
  • optimizeParameters (int (int)) – if 1: nonlinear optimization over al 7 parameters is run (Careful: radius may exceed the given boundaries and then the resulting, considered indices become empty.)
  • probability (float (double)) – the probability of choosing at least one sample free from outliers. [default: 0.99]
Returns:
  • point (float seq. (double*)) – resulting point on axis of symmetrie of cylinder
  • orientationVector (float seq. (double*)) – resulting axis of symmetrie of cylinder
  • radius (float (double)) – resulting fitted radius of cylinder
  • inliers (int (int)) – number of points considered after filtering outliers (due to RANSAC principle)
itom.filter("pclGetHistogram",pointCloudIn, fieldName, minValue, maxValue, steps, histogram[, uniformDistribution])¶

returns the histogram of the specific field.

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • fieldName (str (char*)) – field name, whose values are used for the calculation of the histogram
  • minValue (float (double)) – lower boundary of the uniformly distributed histogram
  • maxValue (float (double)) – upper boundary of the uniformly distributed histogram
  • steps (int (int)) – number of discrete fields in the histogram
  • histogram (dataObject) – The histogram is a int32 data object with size [1 x steps].
Optional parameters:
 

uniformDistribution (dataObject) – provide a dataObject if you want to access the uniform percentage distribution. The resulting data object is of type float32 and has the size [1 x 100]. The value at index j gives the histogram value, where j% of the values lies below that value.
itom.filter("savePolygonMesh",polygonMesh, filename[, type])¶

saves polygonMesh to hard drive (format obj[default], ply, vtk, stl)

Mandatory parameters:
 
  • polygonMesh (polygonMesh) – polygon mesh to save
  • filename (str (char*)) – complete filename (type is either read by suffix of filename or by parameter ‘type’
Optional parameters:
 

type (str (char*)) – type (‘obj’ [default],’ply’,’vtk’,’stl’)
itom.filter("pclGetMinMax3D",pointCloudIn)¶

get the minimum and maximum values on each of the 3 (x-y-z) dimensions in a given pointcloud.

Mandatory parameters:
 

pointCloudIn (pointCloud) – Valid input point cloud
Returns:
  • minXYZ (float seq. (double*)) – minimum (x,y,z) values
  • maxXYZ (float seq. (double*)) – maximum (x,y,z) values
itom.filter("pclEstimateNormals",pointCloudIn, pointCloudOut[, kSearch, viewPoint])¶

estimates normal vectors to the given input point cloud and returns the normal-enhanced representation of the input point cloud

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid point cloud whose normals should be estimated
  • pointCloudOut (pointCloud) – Output point cloud with estimated normals. The type corresponds to the normal-enhanced type of the input cloud.
Optional parameters:
 
  • kSearch (int (int)) – the number of k nearest neighbors to use for the feature estimation [default: 50]
  • viewPoint (float seq. (double*)) – optional camera view point (if given it must have three entries [x,y,z], [default: not used])
itom.filter("pclTransformAffine",pointCloudIn, pointCloudOut, transform)¶

transforms a point cloud with a given homogeneous transformation matrix (4x4 data object)

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – The affine transform is applied to this point cloud
  • pointCloudOut (pointCloud) – The affine transform is applied to this point cloud
  • transform (dataObject) – 4x4 homogeneous transformation matrix
itom.filter("pclStatisticalOutlierRemoval",pointCloudIn, pointCloudOut, meanK, stdDevMulThresh[, negative, keepOrganized])¶

uses point neighborhood statistics to filter outlier data.

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • pointCloudOut (pointCloud) – Output point cloud with removed NaN values
  • meanK (int (int)) – number of nearest neighbors to use for mean distance estimation
  • stdDevMulThresh (float (double)) – standard deviation multiplier for the distance threshold calculation
Optional parameters:
 
  • negative (int (int)) – 0: the regular filter conditions are applied [default], 1: the inverted conditions are applied
  • keepOrganized (int (int)) – filtered points should be kept and set to NaN [1] or removed (potentially breaking organized structure) [default: 0]
itom.filter("pclPassThrough",pointCloudIn, pointCloudOut, fieldName[, minValue, maxValue, negative])¶

filters a point cloud by giving boundary values to a specific dimension (outside or inside of this field).

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • pointCloudOut (pointCloud) – Output point cloud with removed NaN values
  • fieldName (str (char*)) – valid field name to filter ‘x’,’y’,z’,’intensity’...
Optional parameters:
 
  • minValue (float (double)) – minimum value (default: -FLT_MAX)
  • maxValue (float (double)) – maximum value (default: FLT_MAX)
  • negative (int (int)) – 1: values inside of range will be deleted, else 0 [default]
itom.filter("pclGetPercentageThreshold",pointCloudIn, fieldName, percentage[, sortedValues])¶

returns the threshold value at the percentage value in the sorted values of the specific field.

Mandatory parameters:
 
  • pointCloudIn (pointCloud) – Valid input point cloud
  • fieldName (str (char*)) – field name, whose values are used for the determination of the threshold value.
  • percentage (float (double)) – percentage value [0.0,100.0]
Optional parameters:
 

sortedValues (dataObject) – provide a dataObject if you want to access the sorted values of ‘fieldName’. The output is then a float32 data object of size [1 x n], where n is the number of valid values in the specific field.
Returns:

threshold (float (double)) – threshold value (NaN if point cloud was empty or invalid)
itom.filter("pclMeshTriangulation",meshIn, meshOut)¶

calculates polygon mesh with triangles only. This is based on the ear-clipping algorithm, the complexity is n^3 and it does not handle holes.

Mandatory parameters:
 
  • meshIn (polygonMesh) – Valid polygon mesh
  • meshOut (polygonMesh) – output polygon mesh
