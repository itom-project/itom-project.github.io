DataObjectIO¶

Summary: import or export dataObject from/to several file formats.
Type: Algorithm
License: Licensed under LPGL.
Platforms: Windows, Linux
Author: W. Lyda, M. Gronle, ITO, University Stuttgart

Overview¶

This filter contains two different export- / import-functionalities for dataObjects, image or raw.

Image-export functions converts dataObjects to image data and saves them as common image-formats. uint8 or uint16 are saved as gray-values (8bit or if suppored as 16bit) or if the image format allows color are saved according to the defined color palette. float32 or float64 are saved as gray-values (8bit or if suppored as 16bit) or according to the defined color palette. Therefore the values must be between 0.0 and 1.0. Values outside these borders are clipped. If the image format supports RGBA, invalid values are saved as 00 00 00 FF else as 00 00 00.

Basic export-filter definition: source-obj, filename, palette, ... Basic import-filter definition: dst-obj, filename, chanel-specification, ...

Raw-export functions write/read the data to/from txt-based or binary file formats.

These filters are defined in the plugin:

  1. saveIDO
  2. savePNG
  3. loadAnyImage
  4. saveBMP
  5. loadIDO
  6. loadSDF
  7. saveSDF
  8. savePPM
  9. saveRAS
  10. saveDataObject
  11. saveJPG
  12. savePGM
  13. saveTiff
  14. loadDataObject

Filters¶

Detailed overview about all defined filters:

itom.filter("saveIDO",sourceObject, filename[, headerOnly, tags2Binary])¶

saveObject or only its header as xml-compatible file (ido, idh)

Mandatory parameters:
 
  • sourceObject (dataObject) – Any type of dataObject
  • filename (str (char*)) – Destination filename
Optional parameters:
 
  • headerOnly (int (int)) – If 0 the complete dataObject is saved (data + metadata), else the filter saves only metadata
  • tags2Binary (int (int)) – If 0 the metaData is saved as clear text, else double tags are saved as binary
itom.filter("savePNG",sourceImage, filename, palette[, compression, addAlphaChannel])¶

saveObject as portabel networl graphic via openCV (png)

Mandatory parameters:
 
  • sourceImage (dataObject) – 2D-DataObject of anytype
  • filename (str (char*)) – Destination filename
  • palette (str (char*)) – Color palette name [gray, gray16, ...].
Optional parameters:
 
  • compression (int (int)) – Compression rate, 0: high, 9: low
  • addAlphaChannel (int (int)) – If enabled and in case of floating point values, invalid will have alpha low else high

Optional parameters:
 

ColorElement (str (char*)) – Color element character: asIs (default) | alpha | R | G | B | RGB | ARGB | GRAY

itom.filter("loadSDF",DestinationImage, filename)¶

loads an ascii-based surface data file to a 1D or 2D data object (sdf), version aNIST-1.0

Mandatory parameters:
 
  • DestinationImage (dataObject) – Empty dataObjet
  • filename (str (char*)) – Source file name
itom.filter("saveSDF",sourceImage, filename[, decimalSigns, verticalScale, invalidHandling, invalidValue])¶

saves 1D and 2D dataObject to the ascii surface data file format (sdf), version aNIST-1.0

Mandatory parameters:
 
  • sourceImage (dataObject) – 2D-DataObject of anytype or 3-planes DataObject of uint8
  • filename (str (char*)) – Destination filename
Optional parameters:
 
  • decimalSigns (int (int)) – Number of decimal signs (default: 3). For MountainsMaps reduce total number of digits to 5
  • verticalScale (int (int)) – Power of 10 for the zValue (Default: -3, micrometer), only for floating point objects.
  • invalidHandling (int (int)) – Toggles NaN handling if dataObject is floating-type. 0: Write NaN (Default); 1: Skip Value, 2: Substitut by InvalidValue, 3: Substitute by BAD for MountainsMaps.
  • invalidValue (float (double)) – New value for invalid substitution. Default is 0.0

itom.filter("saveDataObject",sourceImage, filename, Format, bitscaling[, lowLimit, highLimit, Color_Format])¶

saves 1D and 2D dataObject to image formats via QImage (*.xbm *.xpm)

Mandatory parameters:
 
  • sourceImage (dataObject) – 2D-DataObject of anytype or 3-planes DataObject of uint8
  • filename (str (char*)) – Destination filename
  • Format (str (char*)) – Format of the Image according to QImage: [‘QImage::Format_Mono’, ‘QImage::Format_MonoLSB’, ‘QImage::Format_Indexed8’, ‘QImage::Format_RGB32’, ‘QImage::Format_ARGB32’
  • bitscaling (int (int)) – Toggle bit-scaling, 0 = off, 1 = autoscale (default), 2 = userdefined ranges
Optional parameters:
 
  • lowLimit (float (double)) – Lowest limit. Will become 0
  • highLimit (float (double)) – Lowest limit. Will become max<bitdepth>
  • Color_Format (str (char*)) – When DataObject of type float32, their format should be –>Gray<– or –>RGB<–
